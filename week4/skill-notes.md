## Workflow
1. Ask the user for the topic, desired depth, target audience, and output file name.
2. Draft an introduction that explains the intuition before presenting formal equations or definitions.
3. Present the subject formally using proper mathematical notation.
4. Include at least two worked examples with full solutions and at least three practice problems with full solutions.
5. Add a dedicated section describing connections to numerical methods, especially the finite element method if applicable.
6. Ensure all LaTeX notation is enclosed with `$...$` for inline math and `$$...$$` for displayed math.
7. Save the result as a Markdown file in the workspace.

## Quality Criteria
- Explanations are precise and suitable for graduate-level readers.
- Mathematical notation is consistently formatted with `$` or `$$` delimiters.
- Examples are complete and fully worked out.
- Practice problems include full solutions, not just hints.
- The numerical methods section relates the topic to practical computation.
